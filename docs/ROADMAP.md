## 🔥 Phase 7.9 — Hot patches (СРОЧНО, до запуска Senler-рассылки)

Найдены в первом пользовательском тестировании 2026-06-26.

### 7.9.1 Telegram in-app browser — авторизация не работает

**Симптом:** юзер кликает ссылку `flicksee.ru` внутри чата в Telegram → открывается **Telegram in-app browser** (НЕ Mini-App, НЕ системный браузер). В этом окне кнопка «Войти» делает `window.open(t.me/Flicksee_bot?start=login_<token>)` — но юзер уже внутри Telegram, новое окно толком не открывается. И пуш от бота не приходит на эту вкладку (нет polling в фокусе?).

**Почему ломается:**
- `window.Telegram?.WebApp?.initData` НЕ инжектится — это не WebApp
- `window.open` с deep-link в Telegram in-app browser ведёт себя странно (открывает в том же окне, теряет polling)
- Юзер может ничего не нажимать т.к. кнопки overlay-чем-то перекрыты

**Решения:**
1. Детектить in-app browser (user-agent `Telegram/`) → показывать banner «Открой в системном Safari/Chrome» с кнопкой `window.open(href, '_blank')` или meta-redirect
2. Или: использовать `tg://resolve?domain=Flicksee_bot&start=login_X` deep-link — он откроет нативный Telegram chat поверх browser, юзер тапает Start → возвращается, polling догоняет
3. Или: если detect Telegram in-app, скрыть login button и показывать ТОЛЬКО «Открой в полном Telegram» (с tg-deeplink на бота с Mini-App)

### 7.9.2 Производительность + state-recovery

**Симптом:** приложение пару раз падало, юзера выкидывало на splash, свайп начинался с начала.

**Гипотезы:**
- React error boundary отсутствует → любая ошибка в дочернем компоненте обнуляет state
- LocalStorage с `flicksee_swipe_currentIndex` восстанавливается, но если данные карточек не догрузились — индекс мимо
- Yandex Metrika / YouTube iframe + parallax + auto-play видео + RAF полнят main thread

**Что сделать:**
- React error boundaries: оборачивать SwipeContainer / Modals — при ошибке показать «что-то пошло не так, продолжить с того же места»
- Замедлить parallax (throttle requestAnimationFrame до 30fps)
- Lazy-load YouTube iframe только при `isActive`
- Кэшировать TMDB ответы в localStorage с TTL 1ч — мгновенный старт повторного визита
- Уменьшить main bundle (сейчас ~600KB JS) — code-split SearchOverlay, MovieDetailModal, FilterModal

### 7.9.3 WOW-эффект при матче (расширяет 12.1)

Перенесено приоритетом в hot-patch — нужно для виральности первой когорты.

См. **Phase 12 — Viral mechanics** ниже, но **поднять до уровня MVP**: даже без screen recording, in-app overlay на матче должен быть. Это первое что почувствует юзер. Без вау-эффекта матч теряется в новой Telegram-нотификации.

**Minimal MVP:**
- Когда detectMatches на клиенте (через polling matches/unseen-count или WebSocket в будущем) обнаруживает новый матч — fullscreen overlay:
  - Размытие фона
  - Большой постер фильма с zoom-in анимацией
  - Текст «🎬 МАТЧ с {{friendName}}» жирно
  - Конфетти CSS (canvas-confetti либо чистый CSS)
  - 2 кнопки: «Посмотреть карточку» / «Закрыть»
- Время показа: 3-4 секунды до auto-dismiss

## 🟥 Phase 8 — Admin & engagement (НАИВЫСШИЙ ПРИОРИТЕТ)

**Зачем:** без админки нельзя:
1. Понять как растёт продукт (нет аналитики).
2. Связаться с 178 уже собранными юзерами.
3. Точечно делать офферы / собирать обратную связь.

### 8.1 Админ-панель (доступ только владельцу)

- Отдельный роут `/admin` с проверкой `req.user.sub === ADMIN_USER_ID` (env var).
- Дашборд:
  - Всего юзеров (всех типов)
  - Юзеров через бот (`isBotStarted=true`)
  - Юзеров за день / неделю / месяц
  - DAU / WAU / MAU графики
  - Лайков, дизлайков, матчей суммарно
  - Топ-10 лайкнутых фильмов
  - Воронка: visit → swipe → friend → match

### 8.2 Broadcast в бота

- В админке текстовое поле + кнопка «Разослать всем юзерам с isBotStarted=true».
- Поддержка inline-кнопки и preview-картинки.
- Sent/Failed счётчики per-message.
- Rate-limit: 30 сообщений/сек (Telegram limit).
- **Запуск:** объявить 178 Senler-юзерам что сервис открыт.

### 8.3 Рекламные / промо-карточки в свайп-деке

- Новая таблица `PromoCard` в БД: image, title, ctaText, ctaUrl, weight, activeFrom/Until.
- Когда фронт берёт следующую карточку, с вероятностью `WEIGHT_PROMO/(WEIGHT_PROMO+WEIGHT_REAL)` отдать промо.
- UI: бейдж «реклама» сверху-слева карточки. Поведение свайпа — обычное (вправо = клик на CTA, влево = скрыть).
- В админке — CRUD промо-карточек.

### 8.4 NPS / micro-survey

- Раз в сессию (если ещё не отвечал) — карточка опроса в свайп-деке.
- Поля:
  - «Нравится сервис?» — да / нет / по 10-балльной
  - «Порекомендуете?» — да / нет
  - «Будете пользоваться?» — да / нет
  - «Какие фичи хотите?» — свободный текст
- В админке: распределение ответов, средний NPS, лента комментариев.

**Объём:** ~3-4 дня работы.

## 🟧 Phase 9 — Splash & onboarding redesign

Сейчас все юзеры (включая залогиненных) попадают на одну и ту же splash-страницу. Это сбивает с толку — повторному юзеру нужен сразу свайп-дек, а не «начать».

### 9.1 Умный лендинг
- Не-залогинен → splash как сейчас (но улучшенный)
- Залогинен → сразу попадает в свайп-дек (`/swipe`), splash скипается
- Splash доступен через `/about` для маркетинговых ссылок и SEO

### 9.2 Splash → лендинг
- Сейчас splash краткий, но не сильный. Нужен полноценный лендинг для холодного трафика из Директа:
  - Hero (видео фоном вместо постеров?)
  - 3 секции «как это работает» (с анимацией)
  - Social proof (отзывы, кол-во пользователей)
  - FAQ
  - CTA «Начать» появляется sticky при скролле

### 9.3 Видео-демо
- Записать 15-секундное демо свайпа на мобиле — поставить вместо или поверх grid постеров.

**Объём:** ~2 дня.

## 🟦 Phase 11 — Content logic: репит и Anime

### 11.1 Повторный визит — без повторений
- Сейчас при логине свайп-дек начинается с начала.

## 🟪 Phase 12 — Viral mechanics (Match overlay)

**Контекст:** Олег видел иностранный аналог где они сделали in-app match-анимацию, наложили на видео доп. кадры и использовали в виральных роликах. Хочет ту же механику.

### 12.1 Match overlay
- Когда матч случается (оба свайпнули one тайтл) — fullscreen overlay:
  - Большой постер фильма
  - Два аватара пользователей пульсируют по бокам
  - Анимация «🎬 МАТЧ» как в Tinder
  - Поверх — конфетти / неоновые блики
  - 3-секундная задержка → CTA «Идём смотреть» с ссылкой на провайдеров
- Запускается **сразу же**, не дожидаясь push (push идёт асинхронно).

### 12.2 Screen recording API integration
- Кнопка «Записать матч» — использует `MediaRecorder` API на странице матча.
- Скачивается mp4-файл который можно загрузить в Reels / TikTok / Shorts.
- Альтернатива: при матче автоматически создавать share-image (canvas → png) для Stories.

### 12.3 Видео-маркетинг ассеты
- Шаблоны рилсов с overlay-композицией.
- Пресет видосов: «Свайпают», «Матч», «Обоим приходит уведомление».

**Объём:** ~4 дня.

## Принципы при разработке

1. **Не продавать фичи, продавать вайб.** Не «трейлеры со звуком», а «снеки целы, фильм найден».
2. **Сначала Senler-аудитория.** 178 человек = первая когорта для всех новых фич.
3. **Виральность важнее ретеншна на первом этапе.** Каждая фича оценивается через призму «можно ли это снять для рилса».
4. **Никаких лишних регистрационных стен.** WebApp юзеров не отпускать на бот-логин — у нас уже есть initData.
